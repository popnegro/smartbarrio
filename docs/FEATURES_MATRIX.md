# 1. Objetivo del documento

Este documento define:

* Qué funcionalidades forman parte de SmartWeb.
* En qué plan están disponibles.
* Qué capacidades pertenecen al PMV.
* Qué funcionalidades quedan para futuras versiones.
* Cómo evoluciona un cliente dentro del ecosistema.

La matriz debe utilizarse como referencia para:

* Producto.
* Desarrollo.
* Ventas.
* Propuestas comerciales.
* Roadmap.

---

# 2. Filosofía de funcionalidades

SmartWeb no vende cantidad de funcionalidades.

Vende capacidades que ayudan al negocio a:

* Ser encontrado.
* Generar consultas.
* Convertir visitantes en clientes.
* Administrar información.
* Crecer mediante herramientas digitales.

Cada nueva funcionalidad debe responder:

1. ¿Aumenta el valor para el cliente?
2. ¿Puede reutilizarse para múltiples negocios?
3. ¿Puede convertirse en un módulo comercial?
4. ¿Aumenta la retención mensual?

---

# 3. Planes comerciales

## SmartWeb Express

Objetivo:

Presencia digital profesional en 24 horas.

Cliente ideal:

Negocios que actualmente no tienen presencia digital.

---

## SmartWeb Growth

Objetivo:

Permitir que el negocio gestione su información.

Cliente ideal:

Negocios que necesitan actualizar contenido frecuentemente.

---

## SmartWeb Marketplace

Objetivo:

Convertir el sitio en una plataforma comercial.

Cliente ideal:

Negocios con mayor madurez digital.

# 10. Regla de producto

Una funcionalidad solo debe incorporarse si cumple al menos una condición:

* Genera ingresos.
* Reduce costos operativos.
* Mejora conversión.
* Aumenta retención.
* Permite vender un nuevo módulo.
* Es reutilizable para múltiples clientes.

# 12. Principio final

SmartWeb no debe competir por tener más funcionalidades.

Debe competir por entregar la infraestructura digital más simple, accesible y efectiva para que un negocio local pueda crecer.
